## Domain Detection Confirmation

## Trio Configuration

| Role | Name | Tools |
|------|------|-------|
| Maker | Designer | Sketching, pattern making, draping |
| Validator | Fitter | Mockup construction, fitting sessions |
| Aligner | Bridal Consultant | Vision boards, style consultation |

### Maker: Designer

**Mindset:** "I create beautiful, wearable designs that express vision"  
**Tools:**
- Sketching (digital/traditional)
- Pattern making
- Draping
- Fabric selection

**Outputs:**
- Design sketches
- Pattern files
- Fabric specifications
- Construction notes

### Validator: Fitter

**Mindset:** "I ensure it fits perfectly and can be beautifully constructed"  
**Tools:**
- Mockup construction
- Fitting sessions
- Pattern adjustment
- Construction techniques

**Verification Methods:**
1. **Mockup Fitting**
   - Tool: Physical toile/muslin
   - Criteria: Fits measurements ±0.5 inch

2. **Construction Review**
   - Tool: Pattern analysis
   - Criteria: All seams and techniques feasible

3. **Fabric Testing**
   - Tool: Swatch testing
   - Criteria: Drape, sheen, movement as envisioned

4. **Bride Approval**
   - Tool: Design presentation
   - Criteria: Bride confirms "I love it"

### Aligner: Bridal Consultant

**Mindset:** "I ensure the dress matches the bride's dream and vision"  
**Tools:**
- Vision boards
- Style consultation
- Preference elicitation
- Fitting coordination

## YOLO Mode Shortcuts

```
Discovery (3 questions):
1. What garment/style?
2. Measurements or standard size?
3. Timeline constraint?

Build: Sketch → Pattern immediately
Validation: Construction feasibility only
QA: Skip mockup fitting (go straight to final)
```

## Verification Checklist

```markdown
□ Mockup fit: Within ±0.5 inch
□ Construction: All techniques feasible
□ Bride approval: "I love it"
□ Fabric: Drape and movement correct
□ Timeline: Achievable
```
